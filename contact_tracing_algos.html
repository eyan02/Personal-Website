Contact Tracing Algorithmns

On this page I'll talk in-depth about the contact-tracing algorithmns developed. User location data is analyzed to extract contact events between two users. The data is then cleaned, organized, and processed to output actionable insight for users and public health divisions.

For access to the algorithmn github respository, please contact me for more info.

Contact Event Simulation

For the purpose of simulation, x and y coordinate positioning data was used with an associated time stamp for each coordinate.

Example: The simulation shows two people entering the same aisle in a supermarket. Person B (blue) tries to practice social distancing, but ends up near Person A (red) for a brief period of time.

Filtering Raw Location Data

The primary method for interaction-event scanning is the calculation of latitude and longitude points in a radial area such that the diameter between each two points is less than or equal to 6 feet. This allows us to identify sub-sections of geolocation data that may contain a contact event between two people.

Measuring Geographical Distance

A primary focus of the analysis is the distance between the two people. For the simulation, I used Pythagorean theorem. GPS location data is not as simple to process. Cleaning the Data

Next we clean and organize the data for our analysis. After calculating the distance between points, the data is then scanned using a Boolean expression to locate interactions between the users. If a user is within a specified number of feet (6 feet for the simulation, as per the CDC guidelines) the interaction is flagged. Next, we count the number of consecutive data points where an interaction occurred. This allows us to quantify the duration of the interaction and exclude all brief contact events (such as walking past someone). We then apply a minimum consecutive value requirement that filters the events for only events that contain X number of datapoints (or more) such that X can be determined by the data analyst.

User Insight Dashboard

The user dashboard populates with a number of metrics a user may find useful. Including average distance from other users, shortest distance at any given time, and a social distance grade. User metrics can be compared to averages according to age, sex, and other demographic characteristics. Demographic information in tandem with contact events provide risk of infection probability estimates that are more accurate than demographic information alone.

Social distance grade for each user is calculated using average skewedness across the user's contact events. For each contact event, a distribution of contact is created. The distribution is then analyzed for directional skew and categorized based on WHO and CDC social distancing guidelines from A, B, C, D, to F depending on the skew of the distribution from > 1 to < -1 respectively.
